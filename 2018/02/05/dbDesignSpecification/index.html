数据库设计规范

发表于 2018-02-05 | 分类于 规范

数据库设计过程中提倡的一些规范。

一般规则

  1. unsigned，not null, 禁止字段定义出现允许 null 的情况
  2. 不要用非负整形，非负由程序控制 比如把-1存到非负字段里，会变成很大的数，在判断用户积分大小的时候，大数比负值危害更大
  3. 字段都必须有默认值，必须有注释，注释可以方便后续开发人员理解
  4. 字段，表名均小写，避免中文拼音，单词间使用下划线相连
  5. 字段，表名用单数，这样简单，便利，举个例子，你有一个袋子，里面有好多个苹果，你会说这是个苹果袋。但无论里面有0，1，百万个苹果，它依然是个袋子。表也是如此，表名需要描述清楚，表里面包含的对象，而非有多少个数据
  6. 表命名有前缀，默认 utf-8 编码，涉及到emoji表情的表，需要设置为 utf8mb4（字段名正常情况下没有前缀）
  7. 数据库使用 InnoDB 引擎, 不要使用 MyISAM 引擎，InnoDB 支持事务，支持行锁定
  8. 每个表尽量有个自增id作为主键，方便增删改查
  9. 每个表都尽量有创建时间和更新时间
  10. status 状态标记： -2删除 -1审核不通过 0初始状态 1审核通过， 删除操作均为标记删除,即update table set status = -1，很少用到delete, 尽可能的保留数据

字段存储类型的选用

  1. 整数类型，特别小的用 tinyint, 一般用 int，特别大的用 bigint, 默认为0
  2. 小数类型，对于钱等需要确保精度的，用 decimal，对于位数比较短一般精度的，用 float，例如打分,9.6, 7.3
  3. 字符串类型，用 varchar, 默认为空字符串
  4. 文本类型，用 text，比如文章内容
  5. 时间类型，用 datetime, 默认为1000-01-01 00:00:00
  6. 日期类型，用 date, 比如1992-01-12, 默认为1000-01-01
  7. 使用 int(10) 存时间戳
  8. 使用 char(15) 存储 IP，当前 IPV6 还很不普及，暂不考虑
  9. 关于选项(类似两个选项或者三个)，使用数字代替，比如 1 代表啥，2代表啥，因为不是所有的数据库都支持 enum

常用字段的定义

字段命名 字段类型 字段描述
created_at datetime 创建时间
updated_at datetime 更新时间
founder_id int 创建人id
updater_id int 更新人员id
current_time int(10） 当前时间
